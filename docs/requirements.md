# TaskMaster Pro - 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
TaskMaster Pro 是一款面向企业团队的在线任务分派及督办管理系统，旨在帮助团队实现任务的高效分配、跟踪、执行和评估，提升整体工作效率和协同能力。

### 1.2 项目目标
- 实现任务的创建、分配、执行、完成的完整生命周期管理
- 提供定时反馈机制，确保任务执行过程可追踪
- 支持多层级督办体系，保证重要任务得到及时处理
- 提供数据统计与分析功能，辅助决策制定

### 1.3 目标用户
- **企业管理层**: 通过系统监控团队整体任务进度，做出资源调配决策
- **项目经理**: 负责项目任务的分解、分配和整体把控
- **团队成员**: 接收任务、执行任务、提交反馈
- **系统管理员**: 管理系统用户、角色权限、系统配置

### 1.4 项目范围

#### 1.4.1 功能范围
| 模块 | 功能 |
|------|------|
| 用户认证 | 用户注册、登录、密码找回、Token认证 |
| 任务管理 | 任务创建、编辑、删除、查询、状态变更 |
| 任务分派 | 任务指派、任务接受/拒绝、任务转交 |
| 定时反馈 | 反馈周期设置、反馈提醒、反馈记录 |
| 跟踪督办 | 督办任务标记、督办状态跟踪、超时预警 |
| 用户权限 | 角色定义、权限分配、权限验证 |
| 管理后台 | 用户管理、部门管理、系统配置 |
| 数据可视化 | 任务统计、工作量分析、效率报表 |

#### 1.4.2 非功能范围
- 响应时间: 主流程操作响应时间 < 2秒
- 并发支持: 支持至少 500 并发用户
- 数据安全: 敏感数据加密存储，传输使用 HTTPS
- 系统可用性: 目标可用性 99.9%

## 2. 功能需求

### 2.1 用户认证模块

#### 2.1.1 用户注册
- 用户名: 6-20位字母数字组合，唯一性校验
- 密码: 8-20位，需包含大小写字母和数字
- 手机号: 11位有效手机号，用于接收通知
- 邮箱: 有效邮箱地址，用于找回密码
- 注册成功后自动发送欢迎邮件

#### 2.1.2 用户登录
- 支持用户名/邮箱/手机号登录
- 密码错误3次后锁定账号15分钟
- 支持记住登录状态（7天有效）
- 登录成功后返回 JWT Token

#### 2.1.3 密码管理
- 忘记密码: 通过邮箱/手机验证码重置
- 修改密码: 需要验证原密码
- 密码强度检测与提示

#### 2.1.4 会话管理
- JWT Token 有效期: 2小时（access_token）
- Refresh Token 有效期: 7天
- 支持多设备登录管理
- 支持强制下线功能

### 2.2 任务管理模块

#### 2.2.1 任务基本信息
| 字段 | 类型 | 说明 |
|------|------|------|
| task_id | UUID | 任务唯一标识 |
| task_no | VARCHAR(20) | 任务编号，格式: TASK-YYYYMMDD-XXX |
| title | VARCHAR(200) | 任务标题，必填 |
| description | TEXT | 任务详细描述 |
| priority | ENUM | 优先级: urgent/high/medium/low |
| status | ENUM | 状态: draft/pending/accepted/in_progress/pending_review/completed/cancelled |
| category | VARCHAR(50) | 任务分类 |
| tags | JSON | 标签列表 |
| attachments | JSON | 附件列表 |

#### 2.2.2 任务创建
- 支持富文本编辑器编写任务描述
- 支持 Markdown 格式
- 支持附件上传（单个文件≤50MB）
- 支持标签管理
- 支持子任务创建（最多3层嵌套）
- 创建时指定优先级和预计完成时间

#### 2.2.3 任务编辑
- 任务创建者随时可编辑
- 任务执行者在"进行中"状态可更新进度
- 任务完成后仅创建者可编辑
- 所有修改记录变更日志

#### 2.2.4 任务查询
- 支持按状态、优先级、分类、时间范围筛选
- 支持关键词全文搜索
- 支持自定义排序
- 支持分页查询（默认20条/页）
- 支持任务收藏和置顶

#### 2.2.5 任务状态流转
```
draft → pending → accepted → in_progress → pending_review → completed
   ↓       ↓          ↓
   └───→ cancelled ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

### 2.3 任务分派模块

#### 2.3.1 任务指派
- 支持单人多任务指派
- 支持批量指派
- 支持指派给自己
- 指派时设置期望完成时间
- 支持指派说明

#### 2.3.2 任务接收
- 被指派人收到通知
- 支持接受/拒绝任务
- 拒绝时需填写原因
- 拒绝后任务自动回到指派人

#### 2.3.3 任务转交
- 任务执行者可转交任务
- 转交需经原指派人审批
- 支持链式转交（最多3级）
- 转交记录完整保留

#### 2.3.4 协作者管理
- 支持添加任务协作者
- 协作者可查看任务详情
- 协作者不可修改核心信息
- 协作者可提交反馈

### 2.4 定时反馈模块

#### 2.4.1 反馈周期设置
| 周期类型 | 说明 |
|----------|------|
| daily | 每日反馈 |
| weekly | 每周一反馈 |
| custom | 自定义周期（每N天） |
| none | 不需要反馈 |

#### 2.4.2 反馈提醒
- 定时任务自动发送提醒通知
- 提醒时间可配置（默认上午10:00）
- 支持邮件和站内信通知
- 提醒内容包含: 任务标题、截止时间、上次反馈内容

#### 2.4.3 反馈记录
- 支持文字、图片、文件反馈
- 反馈必须包含进度百分比（0-100%）
- 支持反馈内容评论
- 反馈记录不可删除，可编辑修改

#### 2.4.4 反馈查看
- 任务创建者可查看所有反馈
- 任务参与者可查看反馈
- 支持反馈时间线展示
- 支持反馈导出

### 2.5 跟踪督办模块

#### 2.5.1 督办任务标记
- 支持将任务标记为督办任务
- 督办任务需指定督办人
- 督办任务需设置重要程度
- 支持督办备注

#### 2.5.2 督办状态跟踪
| 状态 | 说明 |
|------|------|
| supervising | 督办中 |
| warning | 即将超时 |
| overdue | 已超时 |
| resolved | 已解决 |
| closed | 已关闭 |

#### 2.5.3 超时预警
- 任务即将超时（24小时前）发送预警
- 任务超时后每6小时发送提醒
- 预警自动发送给: 执行人、指派人、督办人
- 支持超时自动升级

#### 2.5.4 督办报表
- 按时完成率统计
- 超时任务排行
- 督办任务处置情况

**职责范围:**
- 系统全局配置管理
- 所有用户管理
- 系统安全监控
- 数据备份与恢复

**权限:**
- 全部权限

**职责范围:**
- 本部门人员管理
- 本部门任务监督
- 本部门绩效考核

**权限:**
- user:read, user:write (本部门)
- task:* (本部门)
- supervise:* (本部门)
- report:view (本部门)

### 4.3 项目经理 (project_manager)
**职责范围:**
- 项目任务规划
- 任务分配与跟踪
- 项目进度汇报

**权限:**
- task:create, task:read, task:update
- task:assign (项目内)
- supervise:read
- report:view (项目内)

### 4.4 普通用户 (user)
**职责范围:**
- 接收并完成任务
- 提交任务反馈
- 参与协作任务

**权限:**
- task:read (已分配任务)
- task:update (已分配任务)

### 4.5 访客 (guest)
**职责范围:**
- 查看已共享任务
- 查看公开报表

**权限:**
- task:read (只读，已共享)
- report:view (只读，公开)

## 5. 业务流程

### 5.1 任务创建流程
```
1. 用户登录系统
2. 进入任务管理模块
3. 点击"新建任务"按钮
4. 填写任务基本信息（标题、描述、优先级等）
5. 设置反馈周期和截止时间
6. 保存为草稿或直接发布
7. 系统发送任务创建成功通知
```

### 5.2 任务分派流程
```
1. 任务创建者选择任务
2. 点击"分派任务"按钮
3. 选择执行人（单选/多选）
4. 设置期望完成时间
5. 添加分派说明（可选）
6. 确认分派
7. 系统通知被分派人
8. 被分派人接受/拒绝任务
9. 接受后任务状态变为"进行中"
```

### 5.3 任务反馈流程
```
1. 系统在设定时间发送反馈提醒
2. 执行人登录系统
3. 进入"我的任务"或点击提醒链接
4. 选择需要反馈的任务
5. 填写反馈内容（进度、问题、计划）
6. 上传相关附件（可选）
7. 提交反馈
8. 系统记录反馈时间线
9. 通知相关人员（可选配置）
```

### 5.4 督办处理流程
```
1. 任务即将超时或已超时
2. 系统自动标记为督办状态
3. 通知执行人、指派人、督办人
4. 督办人登录系统查看详情
5. 督办人选择处理方式：
   - 延期处理（调整截止时间）
   - 转交他人（更换执行人）
   - 关闭任务（标记无效/取消）
   - 继续督办（保持跟踪）
6. 系统记录处理结果
```

### 5.5 任务完成流程
```
1. 执行人完成全部工作
2. 点击"申请完成"按钮
3. 填写完成总结
4. 上传最终交付物
5. 提交验收申请
6. 创建者/指派人审核
7. 审核通过：任务状态变为"已完成"
8. 审核不通过：返回"进行中"，说明原因
9. 完成后触发统计更新
```
